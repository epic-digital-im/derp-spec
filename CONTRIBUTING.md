# Contributing to the DERP Specification

See the full contributing guide at [derp-spec.dev/contributing](https://derp-spec.dev/contributing).

## Quick Start

1. **Issues** for bugs, inconsistencies, or gaps
2. **RFCs** (in `rfcs/`) for substantial changes
3. **Pull requests** for typos and small fixes

## Process

1. Submit proposal (issue or RFC)
2. 30-day public comment period (for RFCs)
3. Working group review
4. Merge or close with explanation
