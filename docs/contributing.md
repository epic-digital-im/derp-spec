# Contributing to the DERP Specification

The DERP spec is open. Anyone can propose changes.

## How to Contribute

**File an issue.** Found a problem, inconsistency, or gap? Open an issue on [GitHub](https://github.com/epic-digital-im/derp-spec/issues). Be specific about the section, the problem, and what you think should change.

**Submit an RFC.** For substantial changes (new requirements, tier modifications, lifecycle changes), write an RFC and submit it as a pull request to the `rfcs/` directory. RFCs follow this structure:

```
rfcs/
  0001-short-title.md
```

Each RFC should include:

- **Summary**: one paragraph describing the change
- **Motivation**: why this change is needed
- **Proposal**: the specific change to the spec
- **Alternatives considered**: other approaches and why they were rejected
- **Impact**: which conformance tiers are affected

**Fix a typo.** Small corrections (typos, broken links, formatting) can go directly as pull requests without an RFC.

## Review Process

1. Proposal submitted (issue or RFC pull request)
2. Public comment period (30 days minimum for RFCs)
3. Working group review
4. Merge or close with explanation

## Code of Conduct

Be constructive. Critique ideas, not people. The spec is better when more perspectives contribute.

## Companion Specifications

The DERP spec is part of a family:

- [SAGA Standard](https://saga-standard.dev) (agent format)
- [Agent Bill of Rights](https://agent-rights.org) (agent policy)

Changes that affect cross-spec compatibility should reference the relevant companion spec.

## Stewardship

[FlowState](https://epicflowstate.ai) is the founding steward for DERP v1.x. Stewardship transfers to the DERP Working Group when established.
